# Domain glossary (CRM)

- **Client**: страхователь/клиент; имеет сделки, полисы, платежи (через сделки), может быть застрахованным лицом.
- **Deal**: основная сделка с клиентом; связывает задачи, документы, заметки, чат, расчеты, полисы и платежи; имеет статус (`open`, `on_hold`, `won`, `lost`). Поле `expected_close` хранит вычисляемый `Крайний срок`, а ручной кандидат хранится отдельно в `manual_expected_close` и меняется через событие сделки.
- **DealEvent**: человекочитаемое бизнес-событие сделки для вкладки `Лента` и объяснения дат; ручные события формата `дата + причина` сохраняются в БД без обязательной привязки к полису или платежу, а события платежей/окончаний полисов строятся из текущих связанных сущностей. Технический `AuditLog` показывается отдельно во вкладке `Журнал`.
- **Quote**: расчет по сделке; связан с типом и страховой компанией.
- **Policy**: страховой полис по сделке; связан с клиентом/застрахованным лицом, типом, компанией, каналом продаж. Legacy-поле `status` сохраняется для совместимости, но новые пользовательские потоки опираются на computed status и `is_renewed`. Для КАСКО дополнительно хранит франшизу, признак официального дилера и риск GAP.
- **Payment**: платеж по полису; содержит суммы и даты, связан с финзаписями. Сделка платежа считается производной от сделки полиса; `Payment.deal` остается денормализованным legacy-полем.
- **FinancialRecord**: финансовая запись дохода/расхода по платежу; может входить в ведомость.
- **Statement**: ведомость доходов/расходов; агрегирует финансовые записи и имеет статус.
- **Task**: задача по сделке; имеет статус, приоритет, сроки, назначение.
- **Note**: заметка/комментарий по сделке, может иметь вложения.
- **Document**: документ по сделке (файл) со статусом распознавания/обработки.
- **ChatMessage**: сообщение чата, связанное со сделкой.
- **Notification**: уведомление для пользователя, с типом и payload.
- **NotificationSettings**: настройки уведомлений пользователя, включая Telegram.
- **TelegramProfile**: привязка пользователя к Telegram.
- **NotificationDelivery**: факт доставки уведомления (дедупликация по event/object/date).
- **InsuranceCompany**: справочник страховых компаний.
- **InsuranceType**: справочник типов страхования.
- **SalesChannel**: канал продаж.
- **User/Role/Permission**: модель пользователей и RBAC.
- **OpenNotebookSession**: сохраненная сессия Open Notebook (knowledge-chat).
