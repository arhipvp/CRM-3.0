# Domain glossary (CRM)

- **Client**: страхователь/клиент; имеет сделки, полисы, платежи (через сделки), может быть застрахованным лицом.
- **Deal**: основная сделка с клиентом; связывает задачи, документы, заметки, чат, расчеты, полисы и платежи; имеет статус (`open`, `on_hold`, `won`, `lost`).
- **Quote**: расчет по сделке; связан с типом и страховой компанией.
- **Policy**: страховой полис по сделке; связан с клиентом/застрахованным лицом, типом, компанией, каналом продаж; имеет статус (`active`, `inactive`, `expired`, `canceled`).
- **Payment**: платеж по полису/сделке; содержит суммы и даты, связан с финзаписями.
- **FinancialRecord**: финансовая запись дохода/расхода по платежу; может входить в ведомость.
- **Statement**: ведомость доходов/расходов; агрегирует финансовые записи и имеет статус.
- **Task**: задача по сделке; имеет статус, приоритет, сроки, назначение.
- **Note**: заметка/комментарий по сделке, может иметь вложения.
- **Document**: документ по сделке (файл) со статусом распознавания/обработки.
- **ChatMessage**: сообщение чата, связанное со сделкой.
- **Notification**: уведомление для пользователя, с типом и payload.
- **NotificationSettings**: настройки уведомлений пользователя, включая Telegram.
- **TelegramProfile**: привязка пользователя к Telegram.
- **NotificationDelivery**: факт доставки уведомления (дедупликация по event/object/date).
- **InsuranceCompany**: справочник страховых компаний.
- **InsuranceType**: справочник типов страхования.
- **SalesChannel**: канал продаж.
- **User/Role/Permission**: модель пользователей и RBAC.
- **OpenNotebookSession**: сохраненная сессия Open Notebook (knowledge-chat).
